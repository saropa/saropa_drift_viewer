# Feature 24: Drift Migration Generator

## What It Does

The schema diff feature (Feature 5) already detects differences between Dart code and runtime schema. This feature takes it further: auto-generate actual Drift migration Dart code — not just raw SQL — with proper `createTable`, `addColumn`, `deleteTable`, and `customStatement` calls ready to paste into a migration file.

## User Experience

1. Run the existing "Schema Diff" command
2. The diff panel now shows an additional section: "Generated Migration"
3. Click "Generate Migration Code" → a new Dart file opens with:

```dart
import 'package:drift/drift.dart';

class Migration4To5 extends Migration {
  @override
  Future<void> up(Migrator m) async {
    // New table: audit_log
    await m.createTable(TableInfo(
      'audit_log',
      columns: [
        GeneratedColumn('id', DriftSqlType.int, requiredDuringInsert: false),
        GeneratedColumn('user_id', DriftSqlType.int, requiredDuringInsert: true),
        GeneratedColumn('action', DriftSqlType.string, requiredDuringInsert: true),
        GeneratedColumn('timestamp', DriftSqlType.dateTime, requiredDuringInsert: true),
      ],
    ));

    // Added column: users.phone_number
    await m.addColumn(users, GeneratedColumn(
      'phone_number', DriftSqlType.string,
      requiredDuringInsert: false,
      defaultValue: const Constant(null),
    ));

    // Removed column: users.legacy_field
    await customStatement('ALTER TABLE "users" DROP COLUMN "legacy_field"');

    // Type change: orders.total (TEXT → REAL)
    // WARNING: Manual migration required for type changes.
    // The following is a template — adjust for your data:
    await customStatement('''
      ALTER TABLE "orders" RENAME TO "orders_old";
    ''');
    await customStatement('''
      CREATE TABLE "orders" (
        "id" INTEGER PRIMARY KEY,
        "user_id" INTEGER NOT NULL,
        "total" REAL NOT NULL
      );
    ''');
    await customStatement('''
      INSERT INTO "orders" ("id", "user_id", "total")
      SELECT "id", "user_id", CAST("total" AS REAL) FROM "orders_old";
    ''');
    await customStatement('DROP TABLE "orders_old"');
  }
}
```

4. The generated code includes:
   - `createTable` for new tables
   - `addColumn` for added columns
   - `customStatement` for column drops (SQLite limitation)
   - Table recreation pattern for type changes (with data migration)
   - Comments for manual review areas
5. User copies the migration into their Drift migration file and adjusts as needed

## Architecture

### Dart Type Mapper

Maps SQLite types back to Drift's `DriftSqlType` enum:

```typescript
const SQL_TO_DRIFT_TYPE: Record<string, string> = {
  'INTEGER': 'DriftSqlType.int',
  'TEXT': 'DriftSqlType.string',
  'REAL': 'DriftSqlType.double',
  'BLOB': 'DriftSqlType.blob',
  'BOOLEAN': 'DriftSqlType.bool',
  'DATETIME': 'DriftSqlType.dateTime',
  'BIGINT': 'DriftSqlType.bigInt',
};

function toDriftType(sqlType: string): string {
  const upper = sqlType.toUpperCase();
  return SQL_TO_DRIFT_TYPE[upper] ?? 'DriftSqlType.string';
}
```

### Migration Codegen

Takes a schema diff result and produces Dart source:

```typescript
interface IMigrationAction {
  type: 'createTable' | 'dropTable' | 'addColumn' | 'dropColumn' | 'changeType' | 'renameTable';
  table: string;
  column?: string;
  oldType?: string;
  newType?: string;
  columns?: IColumnDef[];  // for createTable
  nullable?: boolean;
}

class MigrationCodegen {
  generate(diff: ISchemaDiffResult, fromVersion: number, toVersion: number): string {
    const actions = this._diffToActions(diff);
    const lines: string[] = [];

    lines.push("import 'package:drift/drift.dart';");
    lines.push('');
    lines.push(`// Migration from schema v${fromVersion} to v${toVersion}`);
    lines.push(`// Generated by Saropa Drift Advisor — review before using!`);
    lines.push('');

    // Generate the migration steps
    for (const action of actions) {
      lines.push(...this._generateAction(action));
      lines.push('');
    }

    return lines.join('\n');
  }

  private _generateAction(action: IMigrationAction): string[] {
    switch (action.type) {
      case 'createTable':
        return this._generateCreateTable(action);
      case 'dropTable':
        return this._generateDropTable(action);
      case 'addColumn':
        return this._generateAddColumn(action);
      case 'dropColumn':
        return this._generateDropColumn(action);
      case 'changeType':
        return this._generateTypeChange(action);
      default:
        return [`// TODO: Handle ${action.type} for ${action.table}`];
    }
  }

  private _generateCreateTable(action: IMigrationAction): string[] {
    const cols = action.columns!;
    return [
      `// New table: ${action.table}`,
      `await customStatement('''`,
      `  CREATE TABLE "${action.table}" (`,
      ...cols.map((c, i) => {
        const parts = [`    "${c.name}" ${c.sqlType}`];
        if (c.pk) parts.push('PRIMARY KEY');
        if (!c.nullable && !c.pk) parts.push('NOT NULL');
        if (c.defaultValue !== undefined) parts.push(`DEFAULT ${c.defaultValue}`);
        return parts.join(' ') + (i < cols.length - 1 ? ',' : '');
      }),
      `  )`,
      `''');`,
    ];
  }

  private _generateAddColumn(action: IMigrationAction): string[] {
    const nullable = action.nullable ?? true;
    return [
      `// Added column: ${action.table}.${action.column}`,
      `await customStatement(`,
      `  'ALTER TABLE "${action.table}" ADD COLUMN "${action.column}" ${action.newType}${nullable ? '' : ' NOT NULL DEFAULT \\'\\''}',`,
      `);`,
    ];
  }

  private _generateDropColumn(action: IMigrationAction): string[] {
    return [
      `// Removed column: ${action.table}.${action.column}`,
      `// Note: SQLite < 3.35 does not support DROP COLUMN.`,
      `// Use table recreation if targeting older SQLite versions.`,
      `await customStatement(`,
      `  'ALTER TABLE "${action.table}" DROP COLUMN "${action.column}"',`,
      `);`,
    ];
  }

  private _generateDropTable(action: IMigrationAction): string[] {
    return [
      `// Removed table: ${action.table}`,
      `// WARNING: This will delete all data in "${action.table}"!`,
      `await customStatement('DROP TABLE IF EXISTS "${action.table}"');`,
    ];
  }

  private _generateTypeChange(action: IMigrationAction): string[] {
    return [
      `// Type change: ${action.table}.${action.column} (${action.oldType} → ${action.newType})`,
      `// WARNING: SQLite does not support ALTER COLUMN. Manual migration required.`,
      `// Template: recreate table with new column type and migrate data.`,
      `// See https://sqlite.org/lang_altertable.html#making_other_kinds_of_table_schema_changes`,
    ];
  }
}
```

### Integration with Schema Diff Panel

Add a "Generate Migration" button to the existing schema diff panel:

```typescript
// In schema-diff-panel.ts, add to the HTML:
// <button onclick="postMessage({command:'generateMigration'})">Generate Migration Code</button>

// In the message handler:
case 'generateMigration': {
  const codegen = new MigrationCodegen();
  const dartCode = codegen.generate(this._diff, this._fromVersion, this._toVersion);

  const doc = await vscode.workspace.openTextDocument({
    content: dartCode,
    language: 'dart',
  });
  await vscode.window.showTextDocument(doc, vscode.ViewColumn.Beside);
  break;
}
```

## Wiring in extension.ts

```typescript
context.subscriptions.push(
  vscode.commands.registerCommand('driftViewer.generateMigration', async () => {
    // Run schema diff first
    const diff = await computeSchemaDiff(client);
    if (!diff || diff.matches) {
      vscode.window.showInformationMessage('Schema is up to date — no migration needed.');
      return;
    }

    const fromVersion = await vscode.window.showInputBox({
      prompt: 'Current schema version',
      placeHolder: 'e.g., 4',
      validateInput: v => /^\d+$/.test(v) ? null : 'Enter a number',
    });
    if (!fromVersion) return;

    const toVersion = await vscode.window.showInputBox({
      prompt: 'Target schema version',
      value: String(parseInt(fromVersion) + 1),
    });
    if (!toVersion) return;

    const codegen = new MigrationCodegen();
    const dartCode = codegen.generate(diff, parseInt(fromVersion), parseInt(toVersion));

    const doc = await vscode.workspace.openTextDocument({
      content: dartCode,
      language: 'dart',
    });
    await vscode.window.showTextDocument(doc);
  })
);
```

## Testing

- `migration-codegen.test.ts`:
  - New table → generates `CREATE TABLE` with all columns and constraints
  - Added column → generates `ALTER TABLE ADD COLUMN`
  - Dropped column → generates `ALTER TABLE DROP COLUMN` with SQLite version warning
  - Dropped table → generates `DROP TABLE` with data loss warning
  - Type change → generates warning comment (no auto-migration)
  - Multiple changes → correct ordering (creates before alters, alters before drops)
  - Empty diff → empty output
- `dart-type-mapper.test.ts`: test all SQL → DriftSqlType mappings, unknown type fallback

## Known Limitations

- Generated code uses `customStatement` (raw SQL) — not Drift's type-safe API, because the type-safe API requires compile-time table references we don't have
- Type changes produce only a warning comment — full table recreation template is complex and risky to auto-generate
- No support for index creation/deletion in the migration
- No support for composite primary keys or complex constraints
- SQLite ALTER TABLE limitations mean many operations require table recreation
- Generated code must be manually integrated into the user's migration strategy class
- No version detection — user must provide from/to version numbers
- Foreign key constraints are not preserved in the generated code (user must add them)
