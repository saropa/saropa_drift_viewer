# Feature 44: Constraint Wizard

## What It Does

A visual editor for designing and testing database constraints before writing migration code. Define CHECK, UNIQUE, and NOT NULL constraints through a UI, preview which existing rows would violate them, and generate the Dart migration code to apply them. Test constraints against live data before committing to a migration.

### Generated Migration

```dart
// Generated by Saropa Drift Advisor — Constraint Wizard
// Table: users
// Constraints: 3 new constraints

await customStatement('ALTER TABLE "users" ADD CONSTRAINT '
    '"uq_users_email" UNIQUE ("email")');

await customStatement('ALTER TABLE "users" ADD CHECK '
    '("age" >= 0 AND "age" <= 150)');

// NOTE: SQLite does not support ALTER TABLE ADD NOT NULL.
// Workaround: recreate table with NOT NULL constraint.
```

## Architecture

### Constraint Types

```typescript
type ConstraintKind = 'unique' | 'check' | 'not_null';

interface IConstraintDraft {
  id: string;
  kind: ConstraintKind;
  table: string;
  columns?: string[];         // For UNIQUE
  expression?: string;        // For CHECK
  column?: string;            // For NOT NULL
}

interface IConstraintTestResult {
  constraintId: string;
  valid: boolean;
  violationCount: number;
  violations: IViolation[];   // First 20
}

interface IViolation {
  rowPk: unknown;
  values: Record<string, unknown>;
}
```

### Constraint Validator

Tests proposed constraints against the live data:

```typescript
class ConstraintValidator {
  constructor(private readonly _client: DriftApiClient) {}

  async test(constraint: IConstraintDraft): Promise<IConstraintTestResult> {
    switch (constraint.kind) {
      case 'unique':
        return this._testUnique(constraint);
      case 'check':
        return this._testCheck(constraint);
      case 'not_null':
        return this._testNotNull(constraint);
    }
  }

  private async _testUnique(c: IConstraintDraft): Promise<IConstraintTestResult> {
    const cols = c.columns!.map(col => `"${col}"`).join(', ');
    const sql = `
      SELECT ${cols}, COUNT(*) AS _cnt
      FROM "${c.table}"
      GROUP BY ${cols}
      HAVING _cnt > 1
      LIMIT 20
    `;
    const result = await this._client.sql(sql);
    return {
      constraintId: c.id,
      valid: result.rows.length === 0,
      violationCount: result.rows.length,
      violations: result.rows.map((r: Record<string, unknown>) => ({
        rowPk: r[c.columns![0]],
        values: r,
      })),
    };
  }

  private async _testCheck(c: IConstraintDraft): Promise<IConstraintTestResult> {
    const meta = await this._client.schemaMetadata();
    const table = meta.tables.find(t => t.name === c.table);
    const pkCol = table?.columns.find(col => col.pk)?.name ?? 'rowid';

    const sql = `
      SELECT "${pkCol}" AS _pk, *
      FROM "${c.table}"
      WHERE NOT (${c.expression})
      LIMIT 20
    `;
    const result = await this._client.sql(sql);

    // Get total violation count
    const countSql = `SELECT COUNT(*) AS cnt FROM "${c.table}" WHERE NOT (${c.expression})`;
    const countResult = await this._client.sql(countSql);
    const total = (countResult.rows[0] as { cnt: number }).cnt;

    return {
      constraintId: c.id,
      valid: total === 0,
      violationCount: total,
      violations: result.rows.map((r: Record<string, unknown>) => ({
        rowPk: r._pk,
        values: r,
      })),
    };
  }

  private async _testNotNull(c: IConstraintDraft): Promise<IConstraintTestResult> {
    const meta = await this._client.schemaMetadata();
    const table = meta.tables.find(t => t.name === c.table);
    const pkCol = table?.columns.find(col => col.pk)?.name ?? 'rowid';

    const sql = `
      SELECT "${pkCol}" AS _pk, "${c.column}"
      FROM "${c.table}"
      WHERE "${c.column}" IS NULL
      LIMIT 20
    `;
    const result = await this._client.sql(sql);

    const countSql = `SELECT COUNT(*) AS cnt FROM "${c.table}" WHERE "${c.column}" IS NULL`;
    const countResult = await this._client.sql(countSql);
    const total = (countResult.rows[0] as { cnt: number }).cnt;

    return {
      constraintId: c.id,
      valid: total === 0,
      violationCount: total,
      violations: result.rows.map((r: Record<string, unknown>) => ({
        rowPk: r._pk,
        values: r,
      })),
    };
  }
}
```

### Constraint Code Generator

```typescript
class ConstraintCodegen {
  generateDart(constraints: IConstraintDraft[]): string {
    const lines: string[] = [
      '// Generated by Saropa Drift Advisor — Constraint Wizard',
      `// Table: ${constraints[0]?.table ?? 'unknown'}`,
      `// Constraints: ${constraints.length} new constraint(s)`,
      '',
    ];

    for (const c of constraints) {
      switch (c.kind) {
        case 'unique': {
          const cols = c.columns!.map(col => `"${col}"`).join(', ');
          const name = `uq_${c.table}_${c.columns!.join('_')}`;
          lines.push(
            `await customStatement('CREATE UNIQUE INDEX "${name}" ON "${c.table}"(${cols})');`,
            '',
          );
          break;
        }
        case 'check': {
          lines.push(
            '// NOTE: SQLite does not support ALTER TABLE ADD CHECK.',
            '// To add a CHECK constraint, the table must be recreated.',
            `// CHECK expression: ${c.expression}`,
            '',
          );
          break;
        }
        case 'not_null': {
          lines.push(
            `// NOTE: SQLite does not support ALTER TABLE ALTER COLUMN SET NOT NULL.`,
            `// Column "${c.column}" must be recreated with NOT NULL.`,
            `// Ensure no NULL values exist first.`,
            '',
          );
          break;
        }
      }
    }

    return lines.join('\n');
  }

  generateSql(constraints: IConstraintDraft[]): string {
    const lines: string[] = ['-- Generated by Saropa Drift Advisor — Constraint Wizard', ''];

    for (const c of constraints) {
      switch (c.kind) {
        case 'unique': {
          const cols = c.columns!.map(col => `"${col}"`).join(', ');
          const name = `uq_${c.table}_${c.columns!.join('_')}`;
          lines.push(`CREATE UNIQUE INDEX "${name}" ON "${c.table}"(${cols});`);
          break;
        }
        case 'check': {
          lines.push(`-- CHECK: ${c.expression}`);
          lines.push(`-- (Requires table recreation in SQLite)`);
          break;
        }
        case 'not_null': {
          lines.push(`-- NOT NULL on "${c.column}"`);
          lines.push(`-- (Requires table recreation in SQLite)`);
          break;
        }
      }
      lines.push('');
    }

    return lines.join('\n');
  }
}
```

```typescript
context.subscriptions.push(
  vscode.commands.registerCommand('driftViewer.constraintWizard', async (item?: TableItem) => {
    const table = item?.tableMetadata.name ?? await pickTable(client);
    if (!table) return;

    const meta = await client.schemaMetadata();
    const tableMeta = meta.tables.find(t => t.name === table);
    if (!tableMeta) return;

    ConstraintWizardPanel.createOrShow(
      context.extensionUri, client, tableMeta, constraintValidator, constraintCodegen
```

## Testing

- `constraint-validator.test.ts`:
  - UNIQUE constraint with no duplicates → valid
  - UNIQUE constraint with duplicates → violations listed
  - CHECK constraint satisfied by all rows → valid
  - CHECK constraint violated → rows listed
  - NOT NULL on column with no NULLs → valid
  - NOT NULL on column with NULLs → count and examples
  - Violations capped at 20
- `constraint-codegen.test.ts`:
  - UNIQUE generates `CREATE UNIQUE INDEX` in both Dart and SQL
  - CHECK generates comment (SQLite limitation)
  - NOT NULL generates comment (SQLite limitation)
  - Multiple constraints produce combined output
  - Table and column names properly quoted

## Known Limitations

- SQLite doesn't support `ALTER TABLE ADD CHECK` or `ALTER TABLE ALTER COLUMN SET NOT NULL` — code generator produces comments with workaround guidance instead of executable DDL
- UNIQUE constraint is implemented as a unique index (functionally equivalent in SQLite)
- No composite CHECK constraint support (UI is single-expression only)
- CHECK expression is not syntax-validated before execution — invalid SQL will error at test time
- No undo for applied constraints — must be removed manually
- Violation preview limited to 20 rows
- No support for designing foreign key constraints (already handled by Drift schema definitions)
- Generated Dart code uses `customStatement` — not typed Drift migration syntax
